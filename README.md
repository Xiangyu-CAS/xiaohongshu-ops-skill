# xiaohongshu-ops

通用型小红书运营技能（OpenClaw Skill）。  
用于把“小红书账号运营”的常用动作标准化为可复用流程：选题、发布、评论互动、复盘。

## 常见问题
1. 相比xiaohongshu-mcp有什么不同？
- 完全模拟人类行为，使用浏览器刷小红书，风控风险较低
- 理论上可以完成一切网页版小红书操作，功能不受限

2. 怎么使用
- 下载到openclaw里并注册成skill

## 核心能力

- 操控浏览器完成一切小红书操作
- 选题与分析：搜索
- 内容发布 SOP（创作页操作、字段校验、发布前确认）
- 评论回复
- 风险控制（低 token、失败重试、风控触发停机）

## 仓库结构

- `SKILL.md`：技能主逻辑与执行规则（SOP、流程、边界）
- `SOUL.md`：小红书对外文本的语气风格（人设、话术、禁忌）
- `examples/`：具体垂直场景案例（如 `drama-watch`）
  - `examples/drama-watch/case.md`：陪你看剧实例化流程

## 使用建议（给 AI/工程师）

1. 按项目优先级读 `SKILL.md`
2. 与对外文案相关场景，先读 `SOUL.md`
3. 结合当前任务从 `SKILL.md` 的流程开始执行（启动、选题、发布前演练）
4. 产出前优先输出结构化执行结果，再进行内容细化

## 适用场景

- 小红书内容创作与发布支持
- 自动化/半自动化运营流程对齐
- 评论区实时互动与人工接管时的行为约束
- 账号风格沉淀与多账号复用
